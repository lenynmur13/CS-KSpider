================================================================================
                        AI STUDY GUIDE - IMPLEMENTATION PLAN
================================================================================

PROJECT OVERVIEW
----------------
Build a Python CLI application that transforms learning materials into 
structured study assets (flashcards, quizzes, practice tests, audio summaries).

PHASE 1 SCOPE (MVP - This Implementation)
------------------------------------------
✓ Python CLI with minimal FastAPI option
✓ Ingest folder of files (PPTX, PDF, TXT, MD, MP4/MOV/WEBM)
✓ Extract text from documents, transcribe video/audio
✓ Store in SQLite database
✓ Generate flashcards and quizzes using OpenAI Structured Outputs
✓ Export to JSON, Anki CSV, Markdown

================================================================================
                              DATABASE SCHEMA
================================================================================

TABLE: sources
--------------
- id: INTEGER PRIMARY KEY
- filename: TEXT NOT NULL
- filepath: TEXT NOT NULL
- file_type: TEXT NOT NULL (pptx, pdf, txt, md, video)
- file_hash: TEXT (for deduplication)
- created_at: TIMESTAMP
- status: TEXT (pending, processing, completed, failed)

TABLE: documents
----------------
- id: INTEGER PRIMARY KEY
- source_id: INTEGER FOREIGN KEY → sources.id
- title: TEXT
- raw_text: TEXT NOT NULL
- word_count: INTEGER
- created_at: TIMESTAMP

TABLE: chunks
-------------
- id: INTEGER PRIMARY KEY
- document_id: INTEGER FOREIGN KEY → documents.id
- chunk_index: INTEGER
- content: TEXT NOT NULL
- char_count: INTEGER
- created_at: TIMESTAMP

TABLE: study_sets
-----------------
- id: INTEGER PRIMARY KEY
- document_id: INTEGER FOREIGN KEY → documents.id (nullable for multi-doc sets)
- set_type: TEXT (flashcards, quiz, practice_test)
- title: TEXT
- content_json: TEXT (JSON blob of generated content)
- item_count: INTEGER
- created_at: TIMESTAMP

TABLE: jobs (optional tracking)
-------------------------------
- id: INTEGER PRIMARY KEY
- job_type: TEXT (ingest, generate, export)
- status: TEXT (pending, running, completed, failed)
- details: TEXT (JSON)
- created_at: TIMESTAMP
- completed_at: TIMESTAMP

================================================================================
                              PYDANTIC SCHEMAS
================================================================================

class Flashcard(BaseModel):
    question: str
    answer: str
    tags: list[str] = []
    difficulty: Literal["easy", "medium", "hard"]

class FlashcardSet(BaseModel):
    cards: list[Flashcard]

class QuizOption(BaseModel):
    label: str  # A, B, C, D
    text: str

class QuizQuestion(BaseModel):
    prompt: str
    options: list[QuizOption]
    correct_index: int
    explanation: str

class Quiz(BaseModel):
    questions: list[QuizQuestion]

class PracticeTestQuestion(BaseModel):
    question_type: Literal["multiple_choice", "short_answer", "true_false"]
    prompt: str
    options: list[QuizOption] | None = None  # for MC
    correct_answer: str
    explanation: str
    points: int = 1

class PracticeTest(BaseModel):
    questions: list[PracticeTestQuestion]
    total_points: int

================================================================================
                              CLI COMMANDS
================================================================================

# Ingest files from a directory
python -m study_guide ingest ./materials

# List ingested documents
python -m study_guide list documents
python -m study_guide list chunks --doc <id>
python -m study_guide list sets

# Generate study materials
python -m study_guide generate flashcards --doc <id> [--count 20]
python -m study_guide generate quiz --doc <id> [--count 10]
python -m study_guide generate test --doc <id> [--count 20]

# Export study sets
python -m study_guide export <set_id> --format json
python -m study_guide export <set_id> --format anki
python -m study_guide export <set_id> --format markdown

# Show status
python -m study_guide status

================================================================================
                              COST GUARDRAILS
================================================================================

1. Audio Transcription:
   - Chunk audio into segments ≤ 25MB
   - Use lower bitrate (mono, 64kbps) to reduce file size

2. Text Generation:
   - Max 5 chunks per generation call
   - Max 4000 tokens per response
   - Configurable via environment variables

3. Rate Limiting:
   - Add delays between API calls if needed
   - Track token usage in job metadata
